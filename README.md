# 🎯 DOLGO Verbal System

한국어 UX Writing 친근한 톤 변환 Figma 플러그인

## 📋 주요 기능

### 🔄 자동 톤 변환
- 선택한 Figma 텍스트 노드를 한국어 UX Writing 친근한 톤으로 자동 변환
- 명령형 → 제안형, 격식체 → 친근체 등 체계적인 패턴 적용
- 실시간 미리보기로 변환 결과 확인

### 📊 텍스트 분석
- **문장 스타일 분석**: 명령문/의문문/서술문 자동 감지
- **감정 톤 분석**: 친근함, 전문성, 열정, 재미 등 5가지 톤 분류
- **키워드 추출**: 주요 키워드 빈도 분석 및 추출
- **UX 톤 프로파일**: 종합적인 톤 분석 결과 제공

### 🎨 사용자 친화적 UI
- Shadcn 스타일의 모던한 인터페이스
- 카테고리별 패턴 분류 및 설명
- 실시간 분석 결과 시각화
- 직관적인 버튼 구성

### 명령형 → 제안형
| 원본 | 변환 | 설명 |
|------|------|------|
| 확인해보세요 | 확인하기 | 친근한 어조 |
| 이용해보세요 | 이용하기 | 간결한 표현 |
| 참여해보세요 | 참여하기 | 직접적 유도 |
| 신청해보세요 | 신청하기 | 간편함 강조 |
| 문의해보세요 | 문의하기 | 쉬운 접근 |

### 격식체 → 친근체
| 원본 | 변환 | 설명 |
|------|------|------|
| 됩니다 | 돼요 | 친근한 확인 |
| 있습니다 | 있어요 | 친근한 안내 |
| 없습니다 | 없어요 | 부드러운 부정 |
| 입니다 | 이에요 | 부드러운 설명 |

### 격식적 표현 → 친근한 표현
| 원본 | 변환 | 설명 |
|------|------|------|
| 하시면 | 하면 | 간결한 조건 |
| 하십시오 | 해주세요 | 정중한 요청 |
| 하시기 | 하기 | 간단한 안내 |

### 부정적 표현 → 긍정적 표현
| 원본 | 변환 | 설명 |
|------|------|------|
| 실패했습니다 | 다시 시도해보세요 | 긍정적 제안 |
| 오류가 발생했습니다 | 잠시 후 다시 시도해보세요 | 친근한 안내 |
| 문제가 있습니다 | 도움이 필요하시면 문의해주세요 | 도움 제안 |

### 복잡한 표현 → 간단한 표현
| 원본 | 변환 | 설명 |
|------|------|------|
| 사용자 인증이 필요합니다 | 로그인이 필요해요 | 간단한 설명 |
| 시스템 점검 중입니다 | 잠시만 기다려주세요 | 친근한 안내 |
| 서버에 연결할 수 없습니다 | 인터넷 연결을 확인해주세요 | 해결 방법 제시 |

## 📊 분석 기능

### 감정 톤 분석
- **친근함 (Friendly)**: 함께, 나만의, 여러분, 친구, 가족, 편안, 따뜻
- **전문성 (Professional)**: 서비스, 품질, 전문, 안전, 신뢰, 보장, 관리
- **열정 (Enthusiastic)**: 설렘, 최고, 특가, 대박, 놀라운, 환상, 완벽
- **재미 (Playful)**: 재미, 즐거운, 귀여운, 신나는, 웃음, 놀이
- **중립 (Neutral)**: 기본적인 정보 전달 톤

### 문장 스타일 분석
- **명령문**: 하세요, 해보세요, 하시기, 하십시오 패턴 감지
- **의문문**: 입니까, 습니까, 인가요, 할까요 패턴 감지
- **서술문**: 기본적인 설명이나 안내 문장

### 키워드 분석
- 주요 키워드 자동 추출
- 키워드 빈도 분석
- UX Writing 일관성 제안

## 🚀 설치 및 사용법

### 설치
1. Figma에서 플러그인 메뉴 열기
2. "Browse plugins in Community" 클릭
3. "DOLGO Verbal System" 검색
4. "Install" 클릭

### 사용법
1. **텍스트 분석하기**
   - 변환하고 싶은 텍스트 노드 선택
   - "텍스트 분석하기" 버튼 클릭
   - 톤 프로파일과 변환 미리보기 확인

2. **톤 적용하기**
   - 분석 결과 확인 후 "톤 적용하기" 버튼 클릭
   - 선택한 텍스트가 자동으로 변환됨

3. **모든 텍스트 변환**
   - 페이지의 모든 텍스트를 한 번에 변환
   - "모든 텍스트 변환" 버튼 클릭

## 🛠 개발

### 프로젝트 구조
```
DOLGO Verbal System/
├── code.ts          # 메인 플러그인 로직
├── code.js          # 컴파일된 JavaScript
├── ui.html          # 플러그인 UI
├── manifest.json    # 플러그인 메타데이터
├── package.json     # 의존성 관리
├── tsconfig.json    # TypeScript 설정
└── README.md        # 프로젝트 문서
```

### 기술 스택
- **TypeScript**: 타입 안전한 플러그인 개발
- **Figma Plugin API**: Figma와의 상호작용
- **Shadcn UI**: 모던한 UI 디자인 시스템
- **ESLint**: 코드 품질 관리

### 빌드
```bash
# 의존성 설치
npm install

# TypeScript 컴파일
npx tsc

# 린팅
npx eslint code.ts
```

### 패턴 확장
새로운 변환 패턴을 추가하려면 `code.ts`의 `uxPatterns` 배열에 추가:

```typescript
{
  pattern: "원본패턴",
  replacement: "변환패턴", 
  description: "패턴 설명",
  category: "command" | "ending" | "formal" | "negative" | "complex"
}
```

## 🎯 UX Writing 원칙

### 1. 친근함 (Friendliness)
- 명령형보다는 제안형 사용
- 격식체보다는 친근체 선호
- 사용자와의 거리감 줄이기

### 2. 간결함 (Conciseness)
- 불필요한 어휘 제거
- 핵심 메시지 명확화
- 읽기 쉬운 문장 구성

### 3. 긍정성 (Positivity)
- 부정적 표현을 긍정적으로 전환
- 해결책 제시
- 사용자 동기 부여

### 4. 일관성 (Consistency)
- 동일한 상황에서 일관된 표현 사용
- 브랜드 톤앤매너 유지
- 사용자 예측 가능성 확보

### 5. 전문성 (Professionalism)
- 도메인별 전문 용어 정확한 사용
- 개념 간 명확한 구분
- 전문 기관명 완전한 표기
